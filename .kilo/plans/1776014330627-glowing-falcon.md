# Codebase Bug Report & Fix Plan

## Overview
This document outlines all identified bugs, vulnerabilities, and non-working functionality in the placement platform codebase.

---

## 1. AUTHENTICATION & MIDDLEWARE ISSUES

### 1.1 CRITICAL: Collection Name Mismatch
- **File**: `src/app/api/register/route.ts` uses collection `"User"`
- **File**: `src/lib/auth.ts` uses collection `"users"`
- **Impact**: Newly registered users cannot log in because auth system looks in wrong collection
- **Fix**: Make collection names consistent (use "users" or "User" everywhere)

### 1.2 CRITICAL: Signup Form Missing Required Field
- **File**: `src/app/(auth-group)/signup/page.tsx`
- **Issue**: Signup form collects firstName, lastName, email, phone, password but does NOT send `collegeName` to registration API
- **Impact**: Registration will fail with "Missing required fields" error
- **Fix**: Add collegeName field to signup form

### 1.3 Security: OTP Exposed in Console
- **File**: `src/app/api/auth/otp/send/route.ts:26`
- **Issue**: OTP is logged to console: `console.log(\`OTP for ${email}: ${otp}\`)`
- **Fix**: Remove console.log

### 1.4 Security: OTP Timing Attack
- **File**: `src/app/api/auth/otp/verify/route.ts:28`
- **Issue**: Uses direct string comparison without constant-time comparison
- **Fix**: Use constant-time comparison

## 2. API ROUTES SECURITY & BUGS

### 2.1 CRITICAL: No Authentication Required (Multiple Routes)
| Route | Issue |
|-------|-------|
| `test-login` | Anyone can look up users by email |
| `upload` | No auth required for file uploads |
| `problems/run` | Anyone can execute code |
| `generate-mcq` | No auth for MCQ generation |
| `ai-feedback` | No auth for AI feedback |
| `heygen-token` | No auth for HeyGen tokens |

### 2.2 CRITICAL: Tests DELETE Has No Auth
- **File**: `src/app/api/tests/[id]/route.ts`
- **Issue**: DELETE method has no authentication
- **Impact**: Anyone can delete tests

### 2.3 CRITICAL: Identity Auto-Verification
- **File**: `src/app/api/identity/route.ts`
- **Issue**: Auto-verifies all authenticated users without actual verification

### 2.4 CRITICAL: Hardcoded Company List
- **File**: `src/app/api/placements/apply/route.ts`
- **Issue**: Only allows TCS and Wipro (hardcoded allowlist)
- **Impact**: New companies cannot accept applications

### 2.5 Collection Naming Inconsistency
Multiple routes use inconsistent collection names:
- `User` vs `users`
- `Test` vs `tests`
- `Question` vs `questions`
- `Option` vs `options`
- `Result` vs `results`
- `Problem` vs `problems`

### 2.6 Result Page Query Parameter Issue
- **File**: `src/app/exam/[id]/result/page.tsx`
- **Issue**: Requires `resultId` but test-client redirects with `?score=X&total=Y`
- **Impact**: Users redirected back to dashboard immediately after test
